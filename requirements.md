# 2. 서비스 개요

PathTrip은 사용자가 가고 싶은 장소, 맛집, 카페, 숙소 등을 입력하면  
위치와 이동 시간을 기반으로 효율적인 여행 동선을 자동 추천하는 서비스다.

사용자는 단순히 장소를 저장하는 것이 아니라:

- 어떤 순서로 이동해야 효율적인지
- 어떤 장소를 같은 날에 묶어야 하는지
- 이동 시간을 최소화하려면 어떻게 이동해야 하는지

를 추천받을 수 있다.

또한 사용자는 여행 목적에 따라:

- 맛집 중심 여행
- 관광 중심 여행
- 카페 중심 여행
- 사진 스팟 중심 여행

등 우선순위를 설정할 수 있으며,  
서비스는 해당 우선순위를 기반으로 동선을 자동 생성한다.

---

# 3. 서비스 기획 배경

기존 여행 준비 과정에서는 다음과 같은 문제가 존재한다.

- 여행자가 가고 싶은 장소가 많아질수록 동선이 복잡해진다.
- 이동 시간과 위치를 고려하지 않고 장소를 저장하는 경우가 많다.
- 같은 지역 내 장소를 효율적으로 묶기 어렵다.
- 여행 일정이 짧을수록 비효율적인 이동이 발생한다.
- 사용자가 직접 동선을 계획하는 데 시간이 오래 걸린다.

예를 들어 제주도를 2박 3일 여행한다고 가정했을 때:

- 가고 싶은 관광지
- 먹고 싶은 맛집
- 방문하고 싶은 카페
- 사진 찍고 싶은 장소

등은 많지만,  
직접 효율적인 이동 경로를 설계하는 것은 어렵다.

PathTrip은 이러한 문제를 해결하기 위해:

- 위치 기반 분석
- 이동 시간 계산
- 장소 우선순위
- 여행 목적 기반 추천

을 활용하여 자동으로 여행 동선을 추천한다.

---

# 4. 핵심 목표

- 사용자가 여행 계획을 쉽게 세울 수 있어야 한다.
- 이동 시간을 최소화해야 한다.
- 비슷한 지역의 장소를 효율적으로 묶어야 한다.
- 사용자의 여행 목적을 반영해야 한다.
- 여행 동선을 자동 생성할 수 있어야 한다.
- 사용자가 직접 수정할 수 있어야 한다.

# 5. 사용자 유형

## 비회원

- 회원가입을 할 수 있다.
- 로그인을 할 수 있다.
- 보호된 API는 사용할 수 없다.

## 회원

- 여행 일정을 생성할 수 있다.
- 장소를 추가할 수 있다.
- 자동 동선 추천을 받을 수 있다.
- 자신의 여행 일정만 수정 및 삭제할 수 있다.

## 주요 대상 사용자

- 여행 동선을 짜기 어려운 사람
- 이동 시간을 고려하기 어려운 사람
- 맛집, 관광지, 카페를 효율적으로 묶고 싶은 사람
- 짧은 여행 기간 안에 효율적인 여행을 하고 싶은 사람


# 6. 공통 정책

## 인증 정책

- 로그인한 사용자만 여행 기능을 사용할 수 있다.
- JWT Bearer Token 기반 인증을 사용한다.

## 권한 정책

- 사용자는 자신의 여행만 수정할 수 있다.
- 사용자는 자신의 장소만 삭제할 수 있다.
- 사용자는 자신의 추천 동선만 조회할 수 있다.

## 삭제 정책

- 삭제된 데이터는 기본 조회 결과에 포함하지 않는다.
- 중요 데이터는 soft delete를 우선 고려한다.

## 응답 정책

- 모든 API는 공통 응답 포맷을 사용한다.
- 성공 응답은 `success: true`를 사용한다.
- 실패 응답은 `success: false`, `code`, `message`를 사용한다.

## 시간 정책

- DB에는 UTC 기준으로 저장한다.
- API 응답은 ISO 8601 문자열을 사용한다.
